## 🤖 AI 工程與工具

### The Swagger Trap：為什麼直接把原始 API 餵給 LLM 等於送錢給 OpenAI
[🔗 連結](https://theagenticstack.substack.com/p/the-swagger-trap-why-feeding-raw)
- 直接將 OpenAPI Spec 塞入 LLM 上下文窗口是 2026 年 AI 生態最危險的迷思之一
- LLM 是啟發式引擎而非確定性 API 客戶端，會幻想不存在的參數、陷入無限重試迴圈
- 正確做法是建立「工具抽象層（Tool Abstraction Layer）」，將複雜 API 轉為 LLM 可靠理解的簡化工具介面

### Claude Code Part 7：我用單一提示詞建了兩個自動化 Routine
[🔗 連結](https://www.learnwithmeai.com/p/claude-code-routines-setup)
- Claude Code Routines 讓你設定排程/API 呼叫/GitHub 事件觸發的自動化任務，筆電關機也能持續運作
- 作者示範一個本地、一個雲端 Routine，各只需一條提示詞即完成設定
- 屬於 Claude Code MasterClass 第七篇，從安裝到 Agent 編排的完整系列

### 6 個實用技巧讓你更高效使用 Opus 4.7 搭配 Claude Code
[🔗 連結](https://todatabeyond.substack.com/p/6-practical-tips-to-use-opus-47-in)
- Opus 4.7 強化了編碼、代理工作流與多步驟任務能力，但生產力取決於環境設定
- 六大技巧包括：Auto Mode 減少權限中斷、Recap 恢復長對話上下文、依任務難度調整推理力度、讓 Claude 自我驗證輸出
- `/fewer-permission-prompts` 指令可微調重複性工作流的權限提示

### Claude Cowork 過度連接的隱藏成本
[🔗 連結](https://www.coworkoperator.com/p/the-connector-budget-claude-cowork)
- Anthropic 桌面代理 Cowork 已在 macOS/Windows 正式上線，新增分析 API、OpenTelemetry 與企業級 RBAC
- MCP 連接器已超過 75 個，但每多啟用一個就增加上下文消耗、信任風險與預算壓力
- 作者提出「連接器預算（Connector Budget）」概念：每個工作流應限制可觸及的工具數量，避免過度設定反噬效率

### Slack 對分散式 OpenClaw 部署變得更脆弱了
[🔗 連結](https://openclawunboxed.com/p/slack-got-more-fragile-for-distributed)
- Slack 收緊了非 Marketplace 商業分發應用的 API 限制：conversations.history 降至每分鐘 1 次請求、15 物件上限
- 內部自建應用不受影響（50+ 請求/分鐘），但代理商/SaaS 封裝部署受衝擊最大
- 近期 OpenClaw 出現 Socket Mode 連接成功但事件不抵達、v2026.4.2 無法載入 Slack plugin 等問題
