## 🛠 AI 工具與工作流

### The Custom MCPs Included in Your Build to Launch Subscription
[🔗 連結](https://buildtolaunch.substack.com/p/custom-mcps-claude-code-cursor)

- 作者公開其付費訂閱者可用的完整 MCP 工具組，涵蓋研究、Substack 寫作/Notes、Gmail 多帳號管理與跨平台發佈
- 核心理念：MCP 是在 Claude 中消除「切換分頁、重複貼上、每次重建工作流」摩擦的中介層
- 提供一張速查表，讓使用者依自身痛點決定優先安裝哪個 MCP

### The project graph mistake Claude Cowork users will most likely make next
[🔗 連結](https://www.coworkoperator.com/p/the-project-graph-mistake-claude)

- 最常見錯誤：把所有檔案、指令、記憶全塞進單一 Cowork 專案，誤以為它是「公司大腦」
- 正確心智模型：Cowork 專案是「範圍明確的本地執行面」，應作為更大「專案圖譜」中的一個節點
- 目前限制包括僅限桌面本地端、無雲端同步、不支援 Claude Code 與團隊共享

### How to stop hitting Claude usage limits
[🔗 連結](https://ruben.substack.com/p/how-to-stop-hitting-claude-usage)

- Claude 每次回覆都會重新讀取整段對話歷史，因此越長的對話每則訊息成本越高
- 提供 23 個省 token 習慣，從最少人知道的到最顯而易見的排序
- 核心策略：開新對話、精簡前文、避免浪費 token 在不必要的重複上

### Use Codex Inside Claude Code Instead of Choosing Between Models
[🔗 連結](https://todatabeyond.substack.com/p/use-codex-inside-claude-code-instead)

- 停止「哪個 AI 寫程式比較好」的比較思維，改為讓不同工具各司其職：Claude Code 負責撰寫，Codex 負責審查
- Codex 插件提供三種核心指令：Review、Adversarial Review、Rescue（救援卡住的任務）
- AI 編碼工作流正在演變成類似團隊協作模式：一個系統寫、另一個系統 review
